## 方式一：使用启动脚本（推荐 ⭐）

### 0️⃣ 安装依赖（首次运行）
```bash
# 方式 A: 使用安装脚本
cd backend
./install-deps.sh

# 方式 B: 手动安装
cd backend
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### 1️⃣ 启动后端
```bash
# 激活 conda 环境
conda activate meta

# 启动后端
cd backend
python start.py
```

### 2️⃣ 启动前端（新开终端）
```bash
./start-frontend.sh
```

### 3️⃣ 访问应用
- 前端: http://localhost:3000
- 后端API: http://localhost:8000
- API文档: http://localhost:8000/docs

默认账号:
- 用户名: `admin`
- 密码: `admin123`

## 方式二：手动启动（更多控制）

### 后端启动

```bash
cd backend

# 激活环境
conda activate meta

# 安装依赖（仅首次）
pip install -r requirements.txt

# 启动方式选择：
# A. 使用 Python 脚本
python start.py

# B. 使用 Shell 脚本
./start.sh

# C. 直接使用 uvicorn
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

### 前端启动

```bash
cd frontend

# 安装依赖（仅首次）
npm install

# 启动方式选择：
# A. npm start
npm start

# B. npm run dev
npm run dev

# C. 使用脚本
./start.sh
```

## 🔧 常见问题

### Q: conda 环境不存在怎么办？
```bash
# 创建 meta 环境
conda create -n meta python=3.11 -y
conda activate meta
cd backend
pip install -r requirements.txt
```

### Q: 端口被占用？
```bash
# 检查端口
lsof -i :8000  # 后端
lsof -i :3000  # 前端

# 修改端口
# 后端: 修改 start.sh 或 start.py 中的端口
# 前端: 修改 vite.config.ts
```

### Q: 前端显示连接后端失败？
确保：
1. 后端已启动并运行在 8000 端口
2. 检查浏览器控制台的错误信息
3. 访问 http://localhost:8000/docs 确认后端正常

### Q: 如何停止服务？
在运行终端按 `Ctrl + C` 即可停止。
