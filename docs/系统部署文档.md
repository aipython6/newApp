# 医院数据管理系统 - 系统部署文档

## 1. 环境要求

### 1.1 操作系统
- 支持: openEuler, CentOS 7+, Ubuntu 20.04+, Debian 11+
- 推荐: openEuler 22.03 LTS

### 1.2 软件要求
- Docker 20.10+
- Docker Compose 2.0+
- (可选) Nginx 1.18+

### 1.3 硬件要求
- CPU: 2核及以上
- 内存: 4GB及以上
- 磁盘: 20GB及以上可用空间

## 2. Docker Compose 部署（推荐）

### 2.1 项目结构

```
traeProject/
├── backend/
│   ├── Dockerfile
│   ├── main.py
│   ├── ...
│   └── requirements.txt
├── frontend/
│   ├── Dockerfile
│   ├── ...
│   └── package.json
├── docker-compose.yml
└── .env (环境变量配置)
```

### 2.2 环境变量配置

在项目根目录创建 `.env` 文件：

```env
# 应用配置
APP_NAME=医院数据管理系统
DEBUG=false
SECRET_KEY=your-secret-key-change-this-in-production-please
ACCESS_TOKEN_EXPIRE_MINUTES=1440

# 外部MySQL数据库配置
MYSQL_HOST=192.168.100.76
MYSQL_PORT=3306
MYSQL_USER=root
MYSQL_PASSWORD=123456
MYSQL_DATABASE=jxks

# LLM配置（可选）
LLM_API_KEY=
LLM_BASE_URL=
```

**重要**: 请务必修改 `SECRET_KEY` 为随机字符串！

### 2.3 Docker Compose 配置文件

创建 `docker-compose.yml`：

```yaml
version: '3.8'

services:
  backend:
    build:
      context: ./backend
      dockerfile: Dockerfile
    container_name: hospital-backend
    restart: always
    ports:
      - "8000:8000"
    environment:
      - APP_NAME=${APP_NAME}
      - DEBUG=${DEBUG}
      - SECRET_KEY=${SECRET_KEY}
      - ACCESS_TOKEN_EXPIRE_MINUTES=${ACCESS_TOKEN_EXPIRE_MINUTES}
      - MYSQL_HOST=${MYSQL_HOST}
      - MYSQL_PORT=${MYSQL_PORT}
      - MYSQL_USER=${MYSQL_USER}
      - MYSQL_PASSWORD=${MYSQL_PASSWORD}
      - MYSQL_DATABASE=${MYSQL_DATABASE}
    volumes:
      - ./backend/data:/app/data
      - ./backend/chroma_db:/app/chroma_db
    networks:
      - hospital-network

  frontend:
    build:
      context: ./frontend
      dockerfile: Dockerfile
    container_name: hospital-frontend
    restart: always
    ports:
      - "3000:80"
    depends_on:
      - backend
    networks:
      - hospital-network

  # 可选：Nginx反向代理
  # nginx:
  #   image: nginx:alpine
  #   container_name: hospital-nginx
  #   restart: always
  #   ports:
  #     - "80:80"
  #     - "443:443"
  #   volumes:
  #     - ./nginx/nginx.conf:/etc/nginx/nginx.conf
  #     - ./nginx/ssl:/etc/nginx/ssl
  #   depends_on:
  #     - frontend
  #     - backend
  #   networks:
  #     - hospital-network

networks:
  hospital-network:
    driver: bridge
```

### 2.4 后端 Dockerfile

在 `backend/` 目录创建 `Dockerfile`：

```dockerfile
FROM python:3.11-slim

WORKDIR /app

# 安装系统依赖
RUN apt-get update && apt-get install -y --no-install-recommends \
    gcc \
    g++ \
    unixodbc-dev \
    libaio1 \
    && rm -rf /var/lib/apt/lists/*

# 复制依赖文件
COPY requirements.txt .

# 安装Python依赖
RUN pip install --no-cache-dir -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple

# 复制应用代码
COPY . .

# 创建数据目录
RUN mkdir -p /app/data /app/chroma_db

# 暴露端口
EXPOSE 8000

# 启动应用
CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000", "--workers", "4"]
```

### 2.3.5 使用宿主机 Miniconda 虚拟环境（可选）

如果您想使用宿主机 `/home/aaa/miniconda3/envs/meta` 下的虚拟环境，提供了两个方案：

#### 方案一：在容器内重建相同的虚拟环境（推荐）
使用默认的 `Dockerfile`，它会在容器内使用 Miniconda 创建名为 `meta` 的虚拟环境，环境与宿主机保持一致。

#### 方案二：直接挂载宿主机虚拟环境
使用 `Dockerfile.host-env` 和 `docker-compose.host-env.yml`：

```bash
# 使用挂载虚拟环境的配置启动
docker-compose -f docker-compose.host-env.yml up -d --build
```

**注意**：方案二可能会遇到系统库兼容性问题，建议使用方案一。

### 2.4 前端 Dockerfile

在 `frontend/` 目录创建 `Dockerfile`：

```dockerfile
# 构建阶段
FROM node:18-alpine AS builder

WORKDIR /app

# 复制package文件
COPY package*.json ./

# 安装依赖
RUN npm install --registry=https://registry.npmmirror.com

# 复制源代码
COPY . .

# 构建
RUN npm run build

# 生产阶段
FROM nginx:alpine

# 复制构建产物
COPY --from=builder /app/dist /usr/share/nginx/html

# 复制nginx配置
COPY nginx.conf /etc/nginx/conf.d/default.conf

EXPOSE 80

CMD ["nginx", "-g", "daemon off;"]
```

### 2.6 前端 Nginx 配置

在 `frontend/` 目录创建 `nginx.conf`：

```nginx
server {
    listen 80;
    server_name localhost;
    root /usr/share/nginx/html;
    index index.html;

    # 前端路由
    location / {
        try_files $uri $uri/ /index.html;
    }

    # API代理
    location /api {
        proxy_pass http://backend:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}
```

### 2.7 启动服务

```bash
# 构建并启动
docker-compose up -d --build

# 查看日志
docker-compose logs -f

# 查看服务状态
docker-compose ps
```

### 2.8 停止服务

```bash
# 停止服务
docker-compose down

# 停止并删除数据卷（谨慎使用）
docker-compose down -v
```

## 3. 传统部署方式

### 3.1 后端部署

#### 3.1.1 安装 Python 3.11

```bash
# openEuler/CentOS
yum install -y python311 python311-pip python311-devel

# Ubuntu/Debian
apt update
apt install -y python3.11 python3.11-venv python3.11-dev
```

#### 3.1.2 创建虚拟环境

```bash
cd /opt
mkdir -p hospital-system/backend
cd hospital-system/backend

python3.11 -m venv venv
source venv/bin/activate
```

#### 3.1.3 安装依赖

```bash
pip install --upgrade pip
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

#### 3.1.4 配置环境变量

```bash
cat > .env << EOF
APP_NAME=医院数据管理系统
DEBUG=false
SECRET_KEY=your-secret-key-change-this
ACCESS_TOKEN_EXPIRE_MINUTES=1440
MYSQL_HOST=localhost
MYSQL_PORT=3306
MYSQL_USER=root
MYSQL_PASSWORD=123456
MYSQL_DATABASE=jxks
EOF
```

#### 3.1.5 使用 systemd 管理服务

```bash
cat > /etc/systemd/system/hospital-backend.service << EOF
[Unit]
Description=Hospital Data Management System Backend
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/opt/hospital-system/backend
Environment="PATH=/opt/hospital-system/backend/venv/bin"
ExecStart=/opt/hospital-system/backend/venv/bin/uvicorn main:app --host 0.0.0.0 --port 8000 --workers 4
Restart=always

[Install]
WantedBy=multi-user.target
EOF

# 启动服务
systemctl daemon-reload
systemctl enable hospital-backend
systemctl start hospital-backend
systemctl status hospital-backend
```

## 5. 数据备份与恢复

### 5.1 自动备份脚本

创建 `/opt/backup.sh`:

```bash
#!/bin/bash
BACKUP_DIR=/data/backup
DATE=$(date +%Y%m%d_%H%M%S)

mkdir -p $BACKUP_DIR

# 备份SQLite
cp /opt/hospital-system/backend/data/hospital_system.db $BACKUP_DIR/hospital_system_$DATE.db

# 备份MySQL
mysqldump -h 192.168.100.76 -u root -p123456 jxks > $BACKUP_DIR/jxks_$DATE.sql

# 备份向量数据库
tar -czf $BACKUP_DIR/chroma_db_$DATE.tar.gz -C /opt/hospital-system/backend chroma_db

# 删除7天前的备份
find $BACKUP_DIR -name "*.db" -mtime +7 -delete
find $BACKUP_DIR -name "*.sql" -mtime +7 -delete
find $BACKUP_DIR -name "*.tar.gz" -mtime +7 -delete

echo "Backup completed: $DATE"
```

添加到 crontab:

```bash
crontab -e
# 每天凌晨2点备份
0 2 * * * /opt/backup.sh >> /var/log/hospital-backup.log 2>&1
```

## 6. 故障排查

### 6.1 查看日志

```bash
# Docker方式
docker-compose logs -f backend
docker-compose logs -f frontend

# systemd方式
journalctl -u hospital-backend -f
```

### 6.2 常见问题

**问题1: 后端无法连接MySQL**
- 检查MySQL服务是否启动
- 检查网络连通性: `telnet 192.168.100.76 3306`
- 检查账号密码是否正确

**问题2: 前端无法访问后端API**
- 检查后端服务是否正常启动
- 检查防火墙配置
- 检查CORS配置

**问题3: 同步任务不执行**
- 检查APScheduler是否正常启动
- 查看同步任务配置是否正确
- 检查数据库连接是否正常
